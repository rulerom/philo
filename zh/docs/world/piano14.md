# 冯内古特与鲁迪·格茨：技艺的数字化

> 「我们并非取代大师。我们赋予大师一种能在 20 微秒内发声的声音。」

1952 年，库尔特·冯内古特创作了《自动钢琴》（《乌托邦 14》）。书中，最优秀的车床操作工鲁迪·格茨能够凭听觉判断轴承磨损程度。工程师们将他的双手、双耳、直觉录制到磁带上。

机器成为了鲁迪。
但鲁迪已逝。

**冯内古特警示着危险。**
**我们正在构建可能性。**

区别在于应用的伦理。

## 本质

| 冯内古特 (1952) | Decima-8 (2026) |
| --------------- | --------------- |
| 目标：用机器取代人类 | 目标：通过机器增强人类 |
| 媒介：磁带（唯一） | 媒介：.d8p 文件（可复制、可验证） |
| 背景：反乌托邦，人们失业 | 背景：人们从例行工作中解放 |
| 鲁迪：已故，仅存录音 | 大师：活着，录音是其延续 |

**冯内古特设想了一个技艺被录制以*取代*人类的世界。**
**我们正在构建一个技艺被录制以在人类无法企及之处*复制*的世界。**

## 建筑学术语中的「技艺」是什么

| 直觉概念 | 建筑学类比 (Decima-8) |
| ----------------- | ------------------------------- |
| 交易员的「直觉」 | .d8p 中的稳定模式：对订单簿异常的反应 |
| 车床工的「耳朵」 | 振动检测器：频率 → 决策「该更换轴承了」 |
| 医生的「眼光」 | 影像模式识别：肿瘤/正常 |
| 飞行员的「本能」 | 紧急情况反应：20 微秒，而非 200 毫秒 |

**技艺不是魔法。**
**技艺是通过 10,000 小时练习获得的压缩模式。**

Decima-8 不「创造」技艺。
Decima-8 **数字化**技艺并在需要的地方**部署**它。

## 代码中：从人类到 `.d8p`

```c
// 阶段 1：观察大师
typedef struct {
    uint64_t timestamp;
    Input sensor_data;      // 大师看到/听到的内容
    Output master_decision; // 大师的决策
    uint8_t confidence;     // 置信度 (0..15)
} MasterLog;

// 阶段 2：提取模式
D8PFile extract_mastery(const MasterLog* logs, size_t count) {
    D8PFile personality;

    // 进化搜索：调整权重使得
    // swarm_run() 给出与大师相同的决策
    personality.weights = evolve_weights(logs, count);
    personality.thresholds = calibrate_thresholds(logs, count);
    personality.signature = sign_with_master_key(master_private_key);

    return personality;  // 现在是"master.d8p"
}

// 阶段 3：在大师无法企及之处部署
void hft_trading(const D8PFile* master_trader, MarketData* data) {
    Signal signal = swarm_run(master_trader, data);
    // 从数据到决策 20 微秒
    // 大师在休息。他的「分身」在工作。
    execute_trade(signal);
}
```

**关键点：**

- 大师**未被取代**。大师**获得延续**。
- `.d8p` 由大师的密钥签名 → 真实性得到保证。
- 大师每次使用都获得版税（Store）。
