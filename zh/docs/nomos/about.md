# 架构宣言：为何我们开放（以及为何不）

> 「开放不是教条。开放是工具。

> 我们开放应当共有的。我们控制应当保证质量的。」

## Decima-8 不是「开源项目」。

Decima-8 是**一个具有混合模型的生态系统**，其中开放与控制相互增强。

这不是妥协。这是**一个架构决策**。

## 本质

| 开放的内容 | 受控的内容 | 原因 |
| ----------- | ---------- | ---- |
| 模拟器（Pure C） | IDE | 模拟器 = 标准。IDE = 体验质量 |
| .d8p 规范 | Store（验证） | 格式 = 共有。信任 = 可验证 |
| Philosophy / Codex | ASIC 路线图 | 知识 = 自由。战略 = 责任 |
| 库（BSL-1.0） | 角色（.d8p）带 PKI | 工具 = 自由。身份 = 签名 |

**不是「半开放」，** 而是**架构上的务实**。

## 我们吸取的教训

| 项目 | 我们采纳的 | 我们摒弃的 |
| ---- | ---------- | ---------- |
| Linux（托瓦兹） | 开放核心，贡献者精英管理 | 完全去中心化（关键系统需要责任制） |
| 比特币（中本聪） | 协议 = 法律，通过验证建立信任 | 匿名性（我们要求签名和责任制） |
| OpenAI / Anthropic | 通过控制保证质量 | 知识封闭（我们开放 Philosophy 和 Codex） |
| RISC-V | 开放规范 = 标准 | 实现碎片化（我们控制参考实现） |

**我们不复制，而是提取原则并使其适应任务**。

## 为何混合？（架构上）

| 问题 | 纯开源 | 纯闭源 | 混合（Decima-8） |
| ---- | ------ | ------ | ---------------- |
| 分发 | ✅ 快速 | ❌ 缓慢 | ✅ 快速（模拟器开放） |
| 质量 | ⚠️ 依赖社区 | ✅ 可控 | ✅ 可控（IDE, Store） |
| 安全 | ⚠️ 「众目睽睽」≠保证 | ✅ 责任制 | ✅ PKI 签名 + 审计 |
| 变现 | ❌ 困难 | ✅ 直接 | ✅ Store, B2B, ASIC |
| 信任 | ✅ 透明 | ⚠️ 「相信我们」 | ✅ 可验证（代码 + 规范） |

**混合不是「两全其美」，而是关键系统唯一可行的方案**。

## 什么开放（以及为何）

### ✅ 模拟器（Pure C）

- 零依赖
- 可在任何地方编译
- 任何人都可以运行、验证、分叉

**为何**：标准必须是可验证的。如果 `.d8p` 在你的硬件上无法运行——这是 bug，不是「特性」。

### ✅ .d8p 规范

- 文件格式已文档化
- PKI 签名是规范的一部分
- 任何人都可以编写自己的加载器

**为何**：身份必须是可移植的。基底是次要的，意图是主要的。

### ✅ Philosophy / Codex

- 所有原则公开访问
- 三种语言（RU / EN / ZH）
- 任何人都可以阅读、引用、辩论

**为何**：知识必须可扩展。隐藏的哲学 = 邪教。

### ✅ 库（libwui, libdecima）

- BSL-1.0 / MIT
- 可用于商业用途
- 可以分叉、修改

**为何**：工具必须是自由的。自由的工具 = 更多采用。

## 什么受控（以及为何）

### 🔒 IDE

- 所有平台的统一 UX
- 与 Store 集成
- 可视化、调试、性能分析

**为何**：体验质量就是信任。如果 IDE「随便做做」——用户会离开。

### 🔒 Store（身份验证）

- 只有签名的 `.d8p` 才能进入 Store
- 审计权重腐败
- 作者版税

**为何**：安全。带有「卡住忆阻器」的身份不应进入生产环境。

### 🔒 ASIC 路线图

- 战略由架构师确定
- MPW 穿梭、晶圆厂选择、规范

**为何**：责任制。ASIC 错误 = 数百万美元和数年延迟。

### 🔒 B2B 试点

- 与客户集成
- 支持、SLA、保证

**为何**：信任。客户支付的不是代码，而是结果和责任制。

## 如何贡献（规则）

```markdown
## 想帮忙？

### ✅ 无需协调即可做的事：
• 修复模拟器中的 bug（PR → review → merge）
• 提出文档改进建议
• 编写教程、示例、评论
• 在你的硬件上测试，报告问题

### ⚠️ 需要协调的事：
• `.d8p` 规范的变更（在 Issues 中讨论）
• 新的通道、激活级别（架构决策）
• 循环中的变更（确定性——不可协商）

### ❌ 不接受的内容：
• 破坏确定性的变更
• 对第三方库的依赖（零依赖——原则）
• 没有测试的代码（基准测试——强制要求）
```

**这不是「拒绝贡献」。我们只是按原则过滤：「不损害架构」。**

## 变现：为何这是道德的

| 批评 | 回应 |
| ---- | ---- |
| 「开放了代码，但 IDE 收费」 | IDE 对个人项目免费。B2B——收费（支持、SLA）。 |
| 「Store 是交易灵魂」 | Store 是专业知识市场。作者获得版税，用户——经过验证的身份。 |
| 「ASIC 会封闭吗？」 | 规范开放。实现——需要投资。投资者要求回报。 |
| 「为何不全部免费？」 | 免费无法扩展。开发资源必须来自某处。 |

**这不是「剥削」，而是一个所有参与者都获胜的可持续模型**：

- **身份作者** → 使用版税
- **用户** → 有保证的已验证身份
- **架构管理** → 生态系统开发资源
- **社区** → 开放模拟器、文档、标准

## 哲学上：信任与控制

| 原则 | 架构上 | 哲学上 |
| ---- | ------ | ------ |
| 开放 | 模拟器、规范、文档 | 知识必须是自由的 |
| 控制 | IDE, Store, ASIC | 质量需要责任制 |
| 信任 | PKI 签名、验证 | 信任≠盲目信仰。信任 = 可验证 |
| 社区 | 贡献者、贡献 | 群体通过贡献而非消费成长 |

**我们不是在建立「邪教」！我们正在构建一个信任存在于架构中而非意识形态中的生态系统**。

## 现实世界类比

| 系统 | 开放 | 受控 | 原因 |
| ---- | ---- | ---- | ---- |
| HTTP / TLS | 协议 | 证书（CA） | 标准 = 共有。信任 = 可验证 |
| USB / PCIe | 规范 | 设备认证 | 兼容性 = 开放。质量 = 认证 |
| RISC-V | ISA | 实现（芯片） | 架构 = 自由。芯片 = 需要投资 |
| Decima-8 | .d8p, 模拟器 | Store, IDE, ASIC | 身份 = 可移植。信任 = 已验证 |

**我们不是「重新发明轮子」，而是将经过验证的原则应用于新领域**。

## 核心洞察

> 没有控制的开放 = 混乱。

> 没有开放的控制 = 独裁。

> 混合 = 可持续性。

**我们不是「选边站」，而是构建一个开放与控制不是敌人而是伙伴的系统**。
