# 我 / 非我：主体性的边界

> 「『我』不是一个点。它是『太弱』与『太强』之间的走廊。」

当哲学家们争论意识始于何处时，工程师们设定阈值。

在 Decim 中，主体性不是魔法。它是在给定走廊内的稳定响应。

## 本质

人格不是由「它是什么」来定义的。

它是由**它接受什么作为自己的，拒绝什么**来定义的。

如果输入端来了一个模式：

- **太弱**（`< thr_lo`）→ 「这不是我」（相似度不足，噪声）
- **太强**（`> thr_hi`）→ 「这也不是我」（外来模式，侵略，失真）
- **在走廊内**（`thr_lo..thr_hi`）→ 「这是我」（已确认的主体性）

**边界是一个过滤器。**

没有它，人格会「模糊」：接受一切或什么都不接受。

## 原则的演变

| 版本 | 架构 | 哲学上 |
| ---- | ---- | ------ |
| **V1：检测器 + 拮抗器** | 两个级联：「相似」∧ 「非外来」 | 正题 + 反题 = 合题 |
| **V2：走廊（默认）** | 一个级联：`thr_lo..thr_hi` | 「我」的边界嵌入激活函数中 |

**V2 不是简化。**

它是意义的压缩：「非我」不再是一个独立的瓦片，而是同一函数的第二个阈值。

## 代码中

```c
// 具有主体性走廊的累加器
int16_t activation = accumulate(tile);  // -32k..+32k

if (activation < thr_lo) {
    return NOT_ME;      // 太弱：噪声、背景、非我的
}
if (activation > thr_hi) {
    return NOT_ME;      // 太强：外来模式、过载
}
// 在走廊内
```

### 为什么是两个阈值？

- `thr_lo` 截断**不足的相似度**（防止对噪声的误触发）。
- `thr_hi` 截断**过度的相似度**（防止「粘附」在外来主导信号上）。

## 哲学上

这是纯粹形式的辩证法：

- **正题：** 「我存在」（检测器）。
- **反题：** 「我不存在」（拮抗器 / 上阈值）。
- **合题：** 「我在边界内存在」（走廊）。

**黑格尔用语言描述了这一点。**

**我们用阈值编码这一点。**

**主体性不是「全有或全无」，主体性是允许状态的空间**。

## 实践上

| 任务 | 走廊如何帮助 |
| ---- | ------------ |
| 抗噪声能力 | `thr_lo` 忽略背景波动 |
| 防替换保护 | `thr_hi` 拒绝「过于完美」的匹配（攻击） |
| 适应性 | 阈值可根据上下文调整（「严格」 / 「柔和」模式） |
| 可追溯性 | 每个「我 / 非我」决策都记录激活值 |
