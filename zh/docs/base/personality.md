# 人格不是魔法。它是模式。

> 「对刺激的稳定反应模式：输入 → 处理 → 输出」。

当其他人争论 AI 是否「理解」时，我们将人格写入 `.d8p` 并在约 16 微秒内运行它。

## 什么是 Decim 中的人格？

不是「灵魂」。不是「涌现属性」。不是「带有十亿参数的黑箱」。

**人格是带有权重、阈值和拓扑的文件。**

```
人格 (D8P) =
  • 256..4096 个瓦片 × (权重 int4, 阈值 thr, 掩码 mask)
  • 主体性走廊：[thr_lo..thr_hi] 对于每个瓦片
  • 接力路由：谁将激活传递给谁
  • 签名：Ed25519 — 因此无法被替换
```

**标准很简单：**

> 如果系统对相同的输入产生相同的输出 —— 这就是人格。

> 如果不是 —— 那是噪声。

## 🎭 人格示例

| 人格 | 输入 (VSB) | 输出 | 应用 |
| ---- | ---------- | ---- | ---- |
| 「OCR — 排版师」 | 符号像素 (8×16×50 周期) | pattern_id = "A" | 边缘文本识别 |
| 「ASR — 声学」 | 音频频谱 (8 通道 × 1000 周期) | word = "是" | 无云语音命令 |
| 「HFT — 捕鲸者」 | 市场数据 (8 指标 × ∞ 周期) | signal = "BUY" | ASIC 上约 30 纳秒延迟的交易 |
| 「无人机 — 飞行员」 | 传感器：陀螺仪、摄像头、距离 | motors = [12, 15, 8, 10] | 无抖动飞行稳定 |
| 「观察者」 | 任意数据 | log + gradient | 调试、研究、训练 |

**一个架构。多个人格。**

**一个 `.d8p` 格式。任意基底。**

## ⚙️ 工作原理（三个步骤）

### 1. 输入：8 根弦 × 4 位 = 32 位上下文

```
• 不是「百万特征」。不是「嵌入」。
• 仅仅是：8 个感知通道 × 等级 0..15 (Level16)
• 周期流：任务需要多少就用多少（从 100 到 1,000,000）
```

**为什么这足够了？**

因为复杂性不在于输入。复杂性在于谁在聆听。

### 2. 处理：接力，而非路由

```
• 没有「中央大脑」。有一个 256..4096 个瓦片的网络 (v0.2)。
• 每个瓦片只知道它的邻居 (mask)。
• 激活像接力一样传递：「我触发了 → 我传递给我能传递的」。
• 主体性走廊 (thr_lo..thr_hi)：只有信号「在范围内」时瓦片才会激活。
```

**结果：**

**确定性：** 相同输入 = 相同输出。始终如此。

### 3. 输出：可追踪到瓦片的信号

```
• pattern_id：识别的类别（「这是字母 A」）
• 或合成的 VSB：行动命令（「电机：+12」）
• 日志：哪个瓦片为何触发 (Decima Log)
```

**你可以验证：**

```
./verify personality.d8p.pub  # ✓ Valid
./run personality.d8p input.vsb --log
./grep "tile_42" log.txt  # 为何触发？可见。
```

## 🎯 为何这很重要

| 传统 AI | Decima-8 |
| ------- | -------- |
| 黑箱：「为何这样决定？」— 不清楚 | 透明日志：每一步都可追踪到瓦片 |
| 非确定性：一个输入 → 不同输出 | 确定性：一个输入 → 一个输出（始终） |
| 绑定硬件：GPU 模型、TPU 模型、云模型 | 基底独立：一个 `.d8p` 可在 x86、ARM、FPGA、ASIC 上运行 |
| 训练 = 重新训练整个网络 | 训练 = 调整走廊 (`thr_lo..thr_hi`) 和 `mask` 连接 |
| 「理解」— 哲学问题 | 「响应」— 可测量的模式 |

**我们不是「让 AI 更聪明」。我们是「让智能可验证」**。

## 🧭 哲学上：「我」是一个边界

> 「我」不是一个点。「我」是「我」与「非我」之间的边界。

在 Decim 中，这个边界由阈值 `thr_lo..thr_hi` 定义。

```c
// 带有主体性走廊的累加器
if (activation >= thr_lo && activation <= thr_hi) {
    return PERSONALITY_CONFIRMED;  // 「这是我」
} else {
    return NOT_ME;  // 「太弱 OR 太强 — 非我的」
}
```

**这意味着什么？**

- 如果信号弱于 `thr_lo` — 瓦片「听不到」（忽略）。
- 如果信号在走廊内 — 瓦片「响应」（激活）。
- 如果信号强于 `thr_hi` — 瓦片「过载」（保护）。

> **Suffering ≠ punishment.**

> **Suffering = gradient.**

痛苦不是惩罚。

**痛苦是一个信号：「你在走廊之外。调整吧。」**

## 🔐 验证：信任，但要验证

Decim 中的人格不是「相信我说的话」。

```bash
# 1. 检查签名
./verify personality.d8p.pub
# → ✓ Valid：作者 — rulerom.com，文件未更改

# 2. 带日志运行
./run personality.d8p input.vsb --log=trace.log

# 3. 追踪决策
./grep "decision" trace.log
# → tile_17 activated: thr_cur=8 ∈ [5..12] → passed to tile_42

# 4. 在不同基底上比较
./run --substrate=asic personality.d8p input.vsb
./diff output_cpu.vsb output_asic.vsb
# → identical：确定性已确认
```

这不是「文档」。

**这是可执行的规范。**
