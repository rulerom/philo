# 确定性即伦理

> 「可预测性不是限制。它是系统对世界做出的承诺。」

在一个概率性「黑箱」的世界里，确定性不再是一个技术特性，而是一个道德准则。

## 本质

当系统做出影响现实世界的决策时（交易所、机器人、医疗），不可预测性 = 不负责任。

| 系统 | 行为 | 后果 |
| ---- | ---- | ---- |
| LLM（概率性） | 一个请求 → 不同答案 | 无法验证，不可信任 |
| Decima-8（确定性） | 一个请求 → 相同答案 | 可追踪、可测试、负责任 |

**确定性是代码中的伦理。**
你承诺：「在相同条件下，系统将做出相同的决策」。
并且你始终遵守这一承诺。

## 为什么零抖动很重要

| 领域 | 抖动的含义 | 错误代价 |
| ---- | ---------- | -------- |
| 高频交易 | 延迟 20 微秒 → 50 微秒 | 交易丢失，亏损 |
| 机器人 | 控制周期「浮动」 | 机器人抖动、损坏、伤人 |
| 医疗 | 不可预测的响应 | 患者生命风险 |
| 自主系统 | 「有时工作」 | 事故、灾难 |

**抖动不是「技术误差」，而是对用户未履行的承诺。**

## 代码中

```c
// Decima 的确定性循环
uint32_t cycle_start = get_cycle_count();

// 1. 输入标准化（始终相同）
VSBFrame frame = encode_frame(batch);

// 2. 群体处理（始终相同）
Signal signal = swarm_run(frame, personality);

// 3. 时间固定（始终可预测）
```

**与 LLM 对比：**

```python
# LLM：一个请求 → 不同答案
response1 = llm.generate("2+2 等于什么？")  # "4"
response2 = llm.generate("2+2 等于什么？")  # "四"
response3 = llm.generate("2+2 等于什么？")  # "取决于上下文..."

# Decima-8：一个请求 → 一个答案
signal1 = swarm.run(frame)  # BUY @ 15:32:01.020
signal2 = swarm.run(frame)  # BUY @ 15:32:01.020
signal3 = swarm.run(frame)  # BUY @ 15:32:01.020
```

## 哲学上

> 「自由意志 ≠ 混乱。自由 = 在可预测边界内的选择。」

确定性并不否定自由。
它提供了一个**信任空间**，在这个空间中自由才有意义。

- **混乱** — 无法预测 → 无法信任。
- **刚性决定** — 可以预测 → 可以信任。
- **走廊中的自由** — 可以预测边界 → 可以结合上下文信任。

**伦理就是你的行为对那些依赖你的人而言是可预测的。**

## 实践上

| 优势 | 如何帮助 B2B |
| ---- | ------------ |
| 验证 | 可以在法庭/监管机构证明：「系统按规范工作」 |
| 测试 | 一次测试 = 永远有效。不需要 1000 次运行来获得信心 |
| 调试 | 缺陷可复现 → 缺陷可修复 |
| 认证 | 确定性 = 医疗、汽车、航空的要求 |
| 信任 | 客户知道：系统在关键时刻不会「即兴发挥」 |

## 对比：为什么 LLM 不适合关键系统

| 标准 | LLM | Decima-8 |
| ---- | --- | -------- |
| 一个请求 → | 不同答案 | 相同答案 |
| 模型版本 → | 行为变化 | 行为稳定 |
| 硬件 → | 速度不同 | 固定周期 |
| 追踪 → | 「为什么这样决定？」— 不清楚 | 每个 tile 都记录日志 |
| 责任 → | 「这是概率模型」 | 「这是规范，这是日志」 |

**对于聊天 — LLM 合适。**

**对于有后果的决策 — 需要确定性。**
