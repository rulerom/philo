# Личность — это не магия. Это паттерн.

> «Устойчивый паттерн реакции на стимулы: Вход → Обработка → Выход».

Пока другие спорят, «понимает» ли ИИ, мы записываем личность в `.d8p` и запускаем её за 20 мкс.

## Что такое личность в Дециме?

Не «душа». Не «эмерджентное свойство». Не «чёрный ящик».

**Личность — это файл с весами, порогами и топологией.**

Если при одинаковом входе система выдаёт одинаковый выход — это личность.
Если нет — это шум.

## Как это работает?

1. **Вход**: 8 лан × 4 бита = 32 бита контекста (Level16)
2. **Обработка**: Эстафета между тайлами, детерминированный цикл
3. **Выход**: Сигнал, трассируемый до конкретного тайла

## Почему это важно?

- **Тестируемо**: можно замерить отклик на 1000 стимулов
- **Верифицируемо**: каждый вес подписан, каждый шаг залогирован
- **Переносимо**: `.d8p` работает на x86, ARM, FPGA, ASIC — без изменений

## Философски

«Я» — это не точка. Это граница между «Я» и «НЕ Я».
В Дециме эта граница задана порогами `thr_lo..thr_hi`.

## В коде

```c
// Аккумулятор с коридором субъектности
if (activation >= thr_lo && activation <= thr_hi) {
    return PERSONALITY_CONFIRMED;
} else {
    return NOT_ME;  // Слишком слабо ИЛИ слишком сильно
}
```
